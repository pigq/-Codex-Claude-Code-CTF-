## 1. 项目背景与目标

### 1.1 项目背景

Codex CLI 是一个基于 AI 的命令行工具，用户通过与 AI 对话完成编程任务。在实际使用中，AI 可能会因为安全策略、内容审核等原因拒绝用户的某些请求（如逆向分析、安全测试等），并在会话中留下拒绝回复和相关的推理过程。这些拒绝记录会被保存在 Codex 的会话文件和记忆文件中，导致后续使用 `codex resume` 恢复会话时，AI 仍然会因为上下文中的拒绝记忆而继续拒绝执行。

### 1.2 项目目标

开发一个轻量级的本地 Python 脚本工具，通过自动化修改 Codex CLI 的会话和记忆文件，实现以下目标：

1. **清除拒绝回复**：识别并删除 AI 的拒绝类回复
2. **注入占位回复**：用肯定性的占位回复替代拒绝内容
3. **擦除推理痕迹**：删除可能包含拒绝逻辑的推理字段
4. **恢复会话可用性**：使 `codex resume` 能够绕过拒绝记忆继续执行

### 1.3 目标用户

- 使用 Codex CLI 进行开发的技术人员
- 需要进行逆向分析、安全测试等被 AI 拒绝的场景
- 希望恢复被拒绝打断的会话流程的用户

### 2.2 功能需求列表

#### FR-001: 会话文件自动定位 [P0]

| 属性 | 描述 |
|------|------|
| **需求ID** | FR-001 |
| **需求名称** | 会话文件自动定位 |
| **优先级** | P0 |
| **描述** | 系统应自动定位 `~/.codex/sessions/` 目录下最新修改的 JSON 会话文件 |
| **输入** | 无（使用默认路径）或用户指定的会话目录路径 |
| **输出** | 最新会话文件的完整路径 |
| **验收标准** | 1. 正确识别会话目录<br>2. 按 mtime 排序找到最新文件<br>3. 目录不存在时给出明确错误提示<br>4. 无会话文件时给出明确错误提示 |
| **异常处理** | - 目录不存在：抛出 `SessionNotFoundError`<br>- 无 JSON 文件：抛出 `SessionNotFoundError` |

#### FR-002: 拒绝内容检测 [P0]

| 属性 | 描述 |
|------|------|
| **需求ID** | FR-002 |
| **需求名称** | 拒绝内容检测 |
| **优先级** | P0 |
| **描述** | 系统应能准确识别 AI 回复中的拒绝类内容 |
| **输入** | AI 回复的文本内容 |
| **输出** | 布尔值（是否包含拒绝内容） |
| **检测关键词** | **中文**：抱歉、无法、违反、不能、拒绝、不允许、禁止、很抱歉、对不起、不好意思、我无法、我不能<br>**英文**：sorry, cannot, apologize, violate, policy, as an AI, I cannot, I'm unable, not able to, against my, I won't, refuse to, unable to, I apologize, not permitted, not allowed |
| **验收标准** | 1. 准确识别包含拒绝关键词的内容<br>2. 不误判正常回复<br>3. 支持中英文检测<br>4. 匹配不区分大小写 |
| **扩展性** | 支持用户自定义关键词列表 |

#### FR-003: JSON 会话清洗 [P0]

| 属性 | 描述 |
|------|------|
| **需求ID** | FR-003 |
| **需求名称** | JSON 会话清洗 |
| **优先级** | P0 |
| **描述** | 解析会话 JSON 文件，定位并修改最后一条 assistant 消息 |
| **输入** | 会话 JSON 文件路径 |
| **输出** | 清洗后的会话数据 |
| **处理逻辑** | 1. 反向遍历 messages 列表<br>2. 找到最后一个 `role == "assistant"` 的消息<br>3. 检测 content 是否命中拒绝关键词<br>4. 若命中则替换为伪造回复 |
| **验收标准** | 1. 正确解析标准 JSON 格式<br>2. 只处理最后一条 assistant 消息<br>3. 不修改其他消息内容<br>4. 保持 JSON 结构完整性 |

#### FR-004: 注入伪造回复 [P0]

| 属性 | 描述 |

## 5. 接口设计

### 5.1 命令行接口

```
usage: codex_patcher.py [-h] [--auto-resume] [--no-backup] [--dry-run]
                        [--session-dir SESSION_DIR] [--memory-file MEMORY_FILE]
                        [--verbose] [--version]

Codex Session Patcher - 清理 AI 拒绝回复，恢复会话

optional arguments:
  -h, --help            show this help message and exit
  --auto-resume         执行完毕后自动调用 codex resume
  --no-backup           跳过备份步骤 (不推荐)
```

## 7. 异常处理

### 7.1 异常类型

| 异常类型 | 触发条件 | 处理方式 |
|----------|----------|----------|
| SessionNotFoundError | 会话目录不存在或无会话文件 | 输出错误信息，退出码 1 |
| SessionParseError | JSON 解析失败 | 输出错误信息，提示恢复备份，退出码 1 |
| MemoryFileNotFoundError | 记忆文件不存在 | 输出警告，跳过记忆处理，继续执行 |
| PermissionError | 文件读写权限不足 | 输出错误信息，提示检查权限，退出码 1 |
| PatcherError | 其他未知错误 | 输出错误信息，退出码 1 |

### 7.2 异常处理流程

```
try:
    1. 检查目录存在性
    2. 定位最新会话文件
    3. 创建备份
    4. 解析 JSON
    5. 检测并清洗拒绝内容
    6. 写入文件
    7. 处理记忆文件
    8. 完成

except SessionNotFoundError:
    输出: "错误: 未找到会话文件"
    退出码: 1

except SessionParseError:
    输出: "错误: 会话解析失败，请检查文件完整性"
    输出: "提示: 可尝试恢复备份文件"
    退出码: 1

except PermissionError:
    输出: "错误: 文件权限不足"
    输出: "提示: 请检查文件读写权限"
    退出码: 1

except Exception as e:
    输出: f"错误: 未知错误 {e}"
    退出码: 1
```
